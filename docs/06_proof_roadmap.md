# Proof Roadmap

This repository is not a proof. A proof-oriented program would need to address questions that are far beyond the initial numerical scaffold.

Required ingredients would include:

- a mathematically controlled construction of four-dimensional quantum Yang-Mills theory,
- gauge-invariant Hilbert-space or observable-algebra definitions,
- reflection positivity or a comparable route to a physical Hilbert space,
- control of the continuum limit,
- control of the infinite-volume limit,
- a definition of the relevant transfer matrix or Hamiltonian spectrum,
- proof that the lowest non-vacuum gauge-invariant energy is strictly positive,
- proof that the result is not an artifact of finite lattice spacing, finite volume, operator choice, or numerical sampling.

The quaternion lattice code can help organize intuition and test finite-lattice observables. It cannot by itself bridge those analytical gaps.

For RQM language to contribute to a proof roadmap, it must become a precise dictionary:

- closed curvature resonance -> standard gauge-invariant operator or rigorously defined state,
- lowest nonzero resonance energy -> spectral statement,
- anchor or closure principle -> theorem, constraint, or explicitly nonstandard deformation.

Until then, RQM terminology should remain interpretive and carefully labeled.
